Motion tokens

All animations use these duration and easing tokens. Always wrapped to respect prefers-reduced-motion.

Responsive behavior

The current viewport breakpoint is shown in the top-right tag. Resize the browser to see how the system adapts.

--bp-xl

What changes at each breakpoint

Below 768px: Multi-column layouts collapse to single column. Display headlines drop one step in the type scale. Tables become stacked cards.
Below 1024px: Persistent sidebar collapses to a drawer. Top bar becomes more compact. Bottom tab bar appears on mobile.
1280px+: Dashboards can spread to multi-pane. Max-content-width caps at 1200px regardless of viewport.

Skills library

Every MD file in /Design-System, what it covers, and what it enforces. Engineers and AI tools load these per task — small focused files trigger more reliably than one mega-doc.

Brand & visual system

mws-design-system.md

The canonical visual system. Color, typography, spacing, motion tokens plus button, link, card, and icon specs and the dark/light theme rules.

Enforces

  • All values are CSS custom properties — never hardcoded hex
  • Border radius is 2px, or 999px for pills — never in between
  • Theme-stable foreground rule: text on pale fills is always navy
  • --color-teal direct only for sidebar & categorical chart series
iconography.md

Phosphor icons only, regular weight, monoline. Sizing scale, accessibility distinctions, animation rules.

Enforces

  • 5 valid sizes: 16, 20, 24, 32, 48px — never in between
  • aria-hidden on decorative icons; aria-label on icon-only buttons
  • Filled or duotone Phosphor variants are forbidden everywhere
application-lockup.md

The one shared app identity: McDermott symbol + divider + app name. Placement, color-by-surface, responsive behavior, and the naming convention.

Enforces

  • Official McDermott symbol with fill="currentColor" — no bespoke per-app logos
  • The divider is the master-brand cue — never omitted in UI
  • Name in Georgia, Title Case, two lines max; symbol-only at ≤360px
imagery-and-illustration.md

Photography direction (editorial, asymmetric), illustration style (2px stroke, max 3 colors), alt text rules.

Enforces

  • No generic stock photography that screams "business stock"
  • Illustrations: 2px stroke, max 3 colors from McDermott palette
  • Alt text describes the information value — never starts with "Image of"

Components & layout

navigation-and-ia.md

Global navigation decision matrix. When to use top bar vs side nav vs bottom tabs vs drawer vs command palette.

Enforces

  • Top bar for shallow IA (≤7 sections); side nav for dashboards
  • Bottom tabs for ≤5 mobile destinations; drawer for everything else
  • Active state must be indicated in two ways (color + weight or rail)
disclosure-surfaces.md

When to reveal content as a modal, side sheet, popover, accordion, or new full page.

Enforces

  • Modal only when user must respond before continuing
  • Side sheet for related content the user might reference alongside
  • Never nest modals; never use a modal where a sheet would do
forms-and-input.md

Field anatomy, validation timing, input types, error message formula.

Enforces

  • Validate on blur (fields), on submit (forms) — never on keystroke
  • Mark optional, not required — no asterisks for required
  • Error format: what + why + how to fix
  • Selected fills use --accent-interactive (blue/teal per theme)
responsive-and-mobile.md

Breakpoints, touch targets, mobile-specific patterns, mobile keyboard handling.

Enforces

  • Touch targets: 44×44 minimum with 8px spacing between
  • Use inputmode and autocomplete attrs; never disable autofill
  • Sidebar collapses to a drawer below 1024px

States & feedback

loading-empty-and-error-states.md

Skeleton timing, three empty-state flavors (zero-data, filtered-to-zero, no-access), error recovery.

Enforces

  • < 100ms: show nothing. 100-500ms: skeleton. > 3s: explain + cancel
  • Filtered-to-zero uses --bg-surface + border, never a pale fill
  • Every error needs at least one recovery action — never strand the user
notifications-and-feedback.md

Toast vs banner vs alert decisions, severity treatments, status badge spec.

Enforces

  • Default to subtle pale fills, not saturated alert colors
  • Errors never auto-dismiss; info/success after 5s
  • Status badges use pale fills; draft/archived = transparent + border

Quality & content

accessibility.md

WCAG 2.2 AA as the floor. Keyboard navigation, focus management, screen readers, ARIA usage.

Enforces

  • ARIA last resort — prefer semantic HTML
  • Color contrast: 4.5:1 body, 3:1 large text and non-text UI
  • Focus rings must be visible — outline:none without replacement is forbidden
  • Color cannot be the sole indicator of state
ux-copy-and-microcopy.md

Voice and tone, button verbs, error formula, capitalization rules, inclusive language.

Enforces

  • Buttons use verb + noun ("Save changes" not "Submit")
  • Headlines in sentence case, never title case
  • Error formula: what happened + why + how to fix
  • No "Oops!", no exclamation marks in errors, no "just"

Data visualization

data-visualization.md

Chart-type selection, color usage, axis rules, table formatting, accessibility for charts.

Enforces

  • No pie charts with > 5 slices; no 3D charts ever
  • Y-axis starts at zero unless explicitly justified
  • Right-align numbers in tables; left-align text
  • Provide a screen-reader equivalent for every chart

AI layer

ai-trust-and-provenance.md

Citations, source attribution, confidence indicators, distinguishing AI from human content.

Enforces

  • Every factual claim must be linkable to its source
  • Use linguistic hedges ("may", "likely") — not numeric confidence scores
  • AI content visually marked, never blends silently with human content
ai-streaming-and-perceived-latency.md

Token streaming visuals, thinking indicators, stop/cancel patterns, ARIA for async output.

Enforces

  • Cursor blink: 1s on, 1s off — never frantic
  • Stop button replaces Send during generation, in the same position
  • ARIA live regions buffer at sentence boundaries (never per-token)
ai-uncertainty-and-errors.md

"I don't know" patterns, refusals, hallucination guardrails, partial-failure recovery.

Enforces

  • Direct uncertainty: "I don't have that" — no excessive apologies
  • Refusals name the category and offer adjacent path; never moralize
  • Always preserve the user's prompt on error
ai-tool-use-and-agency.md

Tool call display, multi-step plans, permission prompts for destructive actions.

Enforces

  • Every tool call appears as a row (collapsed by default)
  • Destructive actions (email, delete, post) require explicit user approval
  • Permission prompts name the specific consequence — never just "Confirm"
ai-feedback-and-correction.md

Thumbs up/down placement, "why this rating" follow-up, regenerate, memory inspection.

Enforces

  • Feedback is never modal — never blocks the next message
  • Down-vote records first, asks for category second
  • Memory must be inspectable, editable, and clearable
ai-prompting-affordances.md

Suggested prompts, slash commands, file upload, command palette, multimodal input.

Enforces

  • Empty state shows 3-5 representative prompts (not full capability list)
  • Slash commands open a filterable popover with keyboard navigation
  • Surface what context the AI can see (avoids "what does it know" anxiety)

Meta

design-skills-blueprint.md

The master blueprint — every skill file in the system, naming conventions, folder structure. Reference for adding new skills.

Enforces

  • kebab-case filenames; noun phrases (not verbs)
  • One topic per file — smaller files trigger more reliably
  • Domain prefix for the AI layer (ai-...)

Reference

Anti-patterns — never do this

The bug classes the system exists to prevent.

Color
Red error text

Alert colors are background fills only. Text on alerts must be navy.

Theme
--text-primary on pale fill

Themed text tokens flip white in dark mode → unreadable on pale backgrounds.

Forms
Asterisk for required

Mark optional, not required. Most fields are required; marking the minority reduces noise.

Forms
Placeholder as label

Disappears on focus, fails for screen readers, breaks autofill.

Disclosure
Nested modals

If a flow needs more than one modal, redesign as multi-step or full page.

Loading
Spinner under 100ms

Causes a visible flash. Show nothing for sub-100ms operations.

Copy
"OK" / "Yes" / "Submit"

Use specific verbs. "Save changes", "Delete 3 files", "Send invitation."

A11y
outline: none with no replacement

The most common a11y bug. Keyboard users need a visible focus ring.

Radius
border-radius: 8px

McDermott uses 2px for everything except pills (999px). Anything in between is wrong.

Buttons
Wrapping button labels

Buttons never wrap. Keep labels short enough to fit on one line.

AI
Numeric confidence scores

"Confidence: 73%" creates false precision. Hedge linguistically instead.

AI
Auto-execute destructive actions

Email, delete, post. Always require explicit user approval per agentic flow.
